Proyecto integrador: "EduConnect: Plataforma Educativa Ágil"

Estimado equipo, Como Product Owner de este proyecto de plataforma educativa, quiero compartir con ustedes la visión que guiará nuestro desarrollo. Hemos identificado la necesidad de construir un sitio web funcional que facilite el acceso a cursos, recursos y actividades para estudiantes, permitiendo también la gestión básica de usuarios mediante funcionalidades de registro y login.

Para estructurar nuestro trabajo, hemos definido un conjunto de historias de usuario que representan las funcionalidades mínimas que nuestro producto debe cumplir. Estas historias abarcan desde la creación de la infraestructura técnica básica (repositorio, estructura de páginas) hasta el desarrollo de módulos específicos que ustedes mismos, como equipo, propusieron basándose en las necesidades de una plataforma educativa moderna.

Cada historia de usuario describe una necesidad concreta desde la perspectiva del usuario final y debe ser entendida como un compromiso de entrega de valor en cada sprint. El objetivo es que, trabajando de manera incremental, vayamos dando forma a una plataforma navegable, intuitiva y que responda a las expectativas de nuestros futuros usuarios. Confío en su creatividad, compromiso y responsabilidad para construir un producto que no solo cumpla los requisitos básicos, sino que refleje su aprendizaje y visión como futuros desarrolladores ágiles. ¡Mucho éxito en este reto y recuerden que estoy disponible para resolver dudas y priorizar las necesidades a medida que avance el proyecto!

Historias de usuario

Código Nombre Descripción
HU01Crear repositorio común en GitHubCrear un repositorio en GitHub donde se centralice todo el código del proyecto. Este repositorio debe ser accesible para todos los integrantes del equipo, permitiendo realizar commits, crear ramas y manejar versiones de manera ordenada durante el desarrollo.
HU02Configurar rama principal y permisosConfigurar la rama main como principal y asignar roles adecuados de acceso a los colaboradores. Esta configuración busca mantener el flujo de trabajo organizado, asegurando que solo miembros autorizados puedan modificar directamente la rama estable.
HU03Crear archivo index.htmlCrear el archivo inicial index.html que sirva como puerta de entrada al sitio web. Este archivo debe contener las referencias principales al resto de páginas y elementos básicos de estructura HTML como el head, body y enlaces de navegación.
HU04Crear página home.html con bienvenidaCrear la página de inicio home.html que funcione como carta de presentación del proyecto. Debe incluir un saludo al visitante, una breve descripción de la aplicación o sitio web, e invitar al usuario a explorar otras secciones mediante enlaces claros.
HU05Crear página de contacto con formularioCrear una página dedicada a la comunicación donde los usuarios puedan dejar sus datos y mensajes. El formulario debe contener campos como nombre, correo electrónico y mensaje, y estructurarse correctamente utilizando etiquetas semánticas de HTML.
HU06Diseñar estructura básica del sitio en CSSCrear y aplicar un archivo style.css que defina la apariencia general del sitio: tipografías, colores, espaciados, márgenes, alineaciones y fondos. El diseño debe ser coherente entre las distintas páginas para brindar una experiencia de usuario agradable.
HU07Implementar menú de navegación entre páginasImplementar un menú de navegación visible y funcional en todas las páginas del sitio. Este menú debe permitir moverse de manera sencilla entre el inicio, la página de contacto y otras secciones relevantes, utilizando etiquetas de enlace (a).
HU08Agregar pie de página con información del equipoDiseñar un pie de página común a todas las páginas del sitio que contenga el nombre del equipo de desarrollo, derechos de autor, año y enlaces rápidos a contacto o redes sociales. Debe presentarse de forma discreta pero visible para el usuario.
HU09Declarar y usar variables en JavaCrear ejemplos prácticos donde se declaren y usen variables en Java para almacenar información relevante, como nombres de usuario, edades, correos electrónicos o contraseñas. Se deben mostrar en consola para comprobar su almacenamiento correcto.
HU10Implementar estructuras condicionales en JavaImplementar estructuras de control de flujo como if, else, y switch en Java para realizar validaciones básicas. Los condicionales deben permitir responder a diferentes entradas del usuario, como validar el acceso o categorizar edades.
HU11Implementar ciclos para procesamiento repetitivo en JavaUsar ciclos for, while y do-while en Java para recorrer estructuras de datos, repetir instrucciones y automatizar procesos. Se debe mostrar la utilidad de los ciclos en contextos prácticos como mostrar listas de registros o validar entradas.
HU12Crear y manipular arreglos en JavaCrear arreglos simples que almacenen múltiples datos como nombres de usuarios registrados o mensajes de contacto. Implementar funciones básicas para agregar, eliminar o mostrar los elementos del arreglo utilizando ciclos y condicionales.
HU13Mostrar elementos de un arreglo en consolaImplementar una funcionalidad que recorra los arreglos creados en Java y muestre sus elementos uno a uno en consola. Se debe asegurar que la salida sea clara, numerada si es posible, para facilitar la visualización de todos los datos almacenados.
HU14Crear formulario básico de registro de usuariosCrear un formulario sencillo que permita a nuevos usuarios registrarse ingresando nombre, correo y contraseña. El formulario debe estar vinculado a eventos de captura de datos para ser procesados posteriormente por funciones de validación.
HU15Validar datos de formulario de registroImplementar una validación de formulario donde se verifique que ningún campo esté vacío y que los datos cumplan reglas básicas como formato de correo electrónico válido y longitud mínima de la contraseña. Mostrar mensajes de error claros al usuario.
HU16Crear formulario de login de usuarioCrear una interfaz de login donde los usuarios ingresen su correo y contraseña. Esta sección debe incluir validaciones de datos y redireccionar exitosamente al usuario a una sección protegida o enviar un mensaje en caso de credenciales incorrectas.
HU17Validar credenciales de login en JavaComparar los datos capturados en el formulario de login con los datos registrados almacenados previamente. Implementar una lógica condicional que permita aceptar el acceso sólo si el correo y la contraseña coinciden con los registros existentes.
HU18Estilizar formularios de registro y login con CSSAplicar estilos CSS a los formularios de registro y login para mejorar su apariencia. Definir tamaños de campo, colores de botones, estados de error (inputs en rojo) y mensajes de validación, para hacerlos más amigables e intuitivos al usuario.
HU19Agregar iconos de redes sociales en el sitioInsertar iconos visibles de redes sociales (como Facebook, Instagram o LinkedIn) en la página principal o pie de página. Los íconos deben estar enlazados a las respectivas plataformas o perfiles oficiales simulados para mejorar la interacción visual.
HU20Publicar el proyecto en GitHub PagesConfigurar GitHub Pages para desplegar el sitio web de manera pública. Se debe garantizar que los enlaces internos funcionen correctamente, el diseño se visualice igual que en local y el enlace final sea compartido al equipo de evaluación.
HU21Crear página de cursos disponiblesCrear una página que muestre la oferta de cursos disponibles en la plataforma educativa. Debe incluir tarjetas o listas con nombre del curso, breve descripción y botón para más detalles, organizados de forma limpia y atractiva.
HU22Crear página de perfil de estudianteDiseñar una página donde los usuarios puedan visualizar y actualizar su información personal como nombre, correo electrónico, cursos inscritos y progreso en su formación. El diseño debe ser amigable y responsivo.
HU23Crear sección de preguntas frecuentes (FAQ)Crear una sección de preguntas frecuentes donde los usuarios puedan encontrar respuestas rápidas sobre el uso de la plataforma, inscripción a cursos, soporte técnico y otros temas importantes. Debe ser clara y fácil de leer.
HU24Crear página de testimonios de estudiantesDiseñar una página que contenga comentarios y testimonios de estudiantes satisfechos, con nombre, curso tomado y opinión breve. Esto busca generar confianza en nuevos usuarios y destacar el impacto de la formación.
HU25Crear página de eventos y talleresCrear una sección donde se publiquen eventos, talleres y charlas relacionadas con la educación ofrecida en la plataforma. Cada evento debe tener fecha, hora, descripción breve y botón para registrarse.
HU26Crear galería de imágenes de actividadesCrear una página que muestre una galería de imágenes de actividades, eventos o clases. Las imágenes deben organizarse de forma atractiva y responsiva, con posibilidad de verlas en tamaño ampliado.
HU27Crear página de noticias educativasDiseñar una sección de noticias donde se publiquen artículos breves o anuncios relevantes sobre el mundo educativo, avances tecnológicos, becas disponibles y nuevas oportunidades para los estudiantes.
HU28Crear página de inscripción a cursosCrear una página que permita a los estudiantes inscribirse directamente en los cursos de su interés. El formulario debe recoger datos como el curso deseado, nombre del estudiante y correo electrónico.
HU29Crear sección de recursos y material de estudioCrear una página que contenga recursos descargables o enlaces externos a material de apoyo: guías, PDFs, videos educativos, entre otros. Los materiales deben organizarse por categoría o curso.
HU30Crear página de contacto con soporte técnicoCrear una página de contacto enfocada específicamente en soporte técnico, donde los usuarios puedan reportar problemas relacionados con el uso de la plataforma, acceso a cursos o errores técnicos, usando un formulario detallado.
